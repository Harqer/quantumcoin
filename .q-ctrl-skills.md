# Q-CTRL Skills Reference

## Overview

Q-CTRL provides AI-powered quantum infrastructure software with three main products:
- **Boulder Opal** — Quantum control R&D toolkit for designing, simulating, and optimizing quantum hardware
- **Fire Opal** — Automated quantum error suppression for algorithm developers
- **Open Controls** — Open-source library of predefined quantum control pulses and sequences

## Boulder Opal

### What It Does
Quantum control infrastructure software for R&D professionals. Python package to:
- Design error-robust quantum logic gates
- Simulate quantum dynamics (closed/open systems, noise)
- Optimize control pulses (model-based and closed-loop)
- Characterize hardware (noise spectroscopy, parameter estimation)
- Automate calibration with AI-driven closed-loop optimization

### Installation & Setup
```bash
pip install boulder-opal
```
```python
import boulderopal as bo
bo.cloud.authenticate("YOUR_API_KEY")
```

### Core Concepts: Computational Graphs
All computations in Boulder Opal are expressed as data flow graphs:
- Represent quantum systems and desired computations
- Support simple arithmetic to complex high-dimensional quantum operations
- Enable simulations and optimizations
- Key types: Pwc (piecewise-constant), Stf (sampleable tensor functions), Tensor, SparsePwc

## Fire Opal

### What It Does
Automated quantum error suppression for algorithm developers and end users:
- AI-driven error suppression applied transparently to quantum circuits
- Makes quantum hardware results meaningful without requiring hardware expertise
- Supports IBM Quantum, IonQ (direct + Amazon Braket), Rigetti
- Algorithms: QAOA, QPE, VQE, variational classifiers, custom circuits
- As circuits grow in complexity, Fire Opal maintains signal quality vs randomized noise

### Installation & Setup
```bash
pip install fire-opal qiskit qiskit-ibm-runtime qctrl-visualizer
```
```python
import fireopal as fo
fo.authenticate_qctrl_account(api_key="YOUR_QCTRL_API_KEY")

# IBM credentials
credentials = fo.credentials.make_credentials_for_ibm_cloud(
    token="YOUR_IBM_TOKEN",
    instance="YOUR_INSTANCE"
)

# IonQ via Braket
credentials = fo.credentials.make_credentials_for_braket(
    aws_access_key_id="...",
    aws_secret_access_key="...",
    aws_session_token="..."
)
```

### Key API Functions
```python
# Execute circuits with error suppression
result = fo.execute(
    circuits=[qc],
    credentials=credentials,
    backend_name="ibm_brisbane"
)

# Estimate expectation values
result = fo.estimate_expectation(
    circuits=[qc],
    operators=[pauli_op],
    credentials=credentials,
    backend_name="ibm_brisbane"
)

# Run QAOA optimization
result = fo.solve_qaoa(
    cost_function=cost_fn,
    credentials=credentials,
    backend_name="ibm_brisbane"
)

# Iterative/variational algorithms
fo.iterate(circuits, credentials, backend_name, callback)
fo.iterate_expectation(circuits, operators, credentials, backend_name, callback)
```

## API References

### Boulder Opal API
- https://docs.q-ctrl.com/references/boulder-opal

#### Graph Module (`boulderopal.graph`)
Core computation engine. Key classes and functions:
- `Graph` — Main computational graph class with 100+ node operations
- `execute_graph` / `run_optimization` / `run_stochastic_optimization` / `run_gradient_free_optimization`
- `Pwc` — Piecewise-constant tensor functions
- `Stf` — Sampleable tensor functions
- `SparsePwc` — Sparse piecewise-constant operators
- `Tensor` — N-dimensional arrays
- `Target` — Optimization targets
- `Adam` — Optimizer

Key Graph Operations:

## Quick Reference — Common Workflows

### 1. Design Robust Single-Qubit Gate (Boulder Opal)
```python
import boulderopal as bo
import numpy as np

graph = bo.Graph()

duration = 100e-9  # 100 ns
segment_count = 64
signal = graph.real_optimizable_pwc_signal(
    segment_count=segment_count,
    duration=duration,
    minimum=-2*np.pi*20e6,
    maximum=2*np.pi*20e6
)

hamiltonian = graph.pwc_operator(signal, operator=graph.pauli_matrix("X")/2)

noise = graph.constant_pwc_operator(
    duration=duration,
    operator=graph.pauli_matrix("Z")/2
)

target = graph.target(operator=np.array([[0, -1j], [-1j, 0]]))
infidelity = graph.infidelity_pwc(
    hamiltonian=hamiltonian,
    target=target,
    noise_operators=[noise]
)

result = bo.run_optimization(
    graph=graph,
    cost_node_name="infidelity",
    output_node_names=["signal"]
)
```

## Key Concepts

### Boulder Opal Core Workflow
1. **Define** — Create a computational graph representing your quantum system
2. **Simulate** — Calculate dynamics under realistic conditions
3. **Optimize** — Find control pulses that maximize gate fidelity
4. **Verify** — Evaluate robustness via filter functions
5. **Export** — Format outputs for hardware controllers
6. **Automate** — Close the loop with AI-driven calibration

### Fire Opal Core Workflow
1. **Build** — Create quantum circuit in Qiskit/OpenQASM
2. **Validate** — Check circuit meets requirements
3. **Execute** — Submit with `fo.execute()` — error suppression applied automatically
4. **Retrieve** — Get corrected measurement results

### Error Suppression vs Error Correction
- **Fire Opal** = Error *suppression* (no extra qubits needed, transparent to user)
- Different from QEC which requires ancilla qubits and syndrome measurements
- Combines: gate compilation, dynamical decoupling, noise-aware transpilation, measurement error mitigation
